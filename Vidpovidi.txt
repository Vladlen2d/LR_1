2.Для чого використовується  команда add? 
Kонтролювання нових файлів(добавлення файлів)
3.Для чого використовується  команда status? 
Перевірка статусу файлів.(Можна дізнатися стан файлів)
4.Для чого використовується  команда commit? 
Створення, внесення коміту.
5.Для чого використовується  команда init? 
Створення репозиторія.
6.Що необхідно писати в коментарях комітів? 
Якщо відбулися якісь зміни.
7.Для чого використовується  команда checkout? 
 Для переходу на іншу гілку.
8.Для чого використовується  команда reset? 
Усвідомлення скидання.
9.Для чого використовується  команда log? 
Перегляд історії комітів.
10.Для чого використовується  команда status? 
Перевірка статусу файлів.(Можна дізнатися стан файлів)
11.Як відновити кілька файлів з попередньо створеного коміта? 
Переглянути стан репозиторію, потім перемістити гілку master до середнього коміту і git reflog.
12.Як створити нову гілку? 
За допомогою команди git branch
13.Як перейти на нову гілку? 
Команда CHECKOUT
14.Як перейти на основну гілку? 
Основна гілка : master. Щоб до неї перейти, треба виконати:
$ git checkout master
Switched to branch 'master'
15.Як виконати злиття гілок? 
Треба перемкнутися на вашу робочу гілку і виконати команду git merge:
$ git checkout master
Switched to branch 'master'
$ git merge iss53
Merge made by the 'recursive' strategy.
index.html |    1 +
1 file changed, 1 insertion(+)
